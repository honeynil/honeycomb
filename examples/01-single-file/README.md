# Пример 1: Single File Application
## Что делает этот пример?

Простой HTTP-сервер с одним эндпоинтом `/som`, который возвращает JSON-ответ.

## Когда использовать?
- мелкие скрипты/утилиты
- прототипы/концепты/учебные примеры
- простые веб-хуки/cli с одной-двумя коммандами
## Структура
```
01-single-file/
├── main.go     # Весь код приложения
├── go.mod      # Зависимости
└── README.md   # Документация
```
## Почему именно так?

**`main.go` в корне** — это идиоматично для Go. Не нужно создавать `cmd/app/main.go` для простого приложения.

**Никаких дополнительных директорий** — следуем принципу "Start Simple". Не создаём структуру "на вырост".

**`go.mod` обязателен** — даже для простейшего проекта. Это современный стандарт Go (с версии 1.11+).
